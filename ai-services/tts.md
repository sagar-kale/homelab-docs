# Text-to-Speech

Two TTS engines run side-by-side. The Vision Agent lets you switch between them per session from the UI.

---

## Kokoro TTS

- **Model:** Kokoro-82M (ONNX)
- **Runtime:** CPU-only, Docker container
- **API:** OpenAI-compatible (`POST /v1/audio/speech`)
- **Default voice:** `af_bella` — warm, natural American English female
- **Latency:** ~0.5s per phrase
- **Best for:** Fast conversational responses

## Chatterbox Turbo

- **Model:** [ResembleAI/chatterbox-turbo](https://github.com/resemble-ai/chatterbox) — 2 diffusion steps vs 10 in standard Chatterbox
- **Runtime:** ROCm GPU (Radeon 8060S), native Python + systemd
- **Voice:** Cloned from a Kokoro af_bella reference WAV — same voice persona, more expressiveness
- **Latency:** ~2.1–2.4s per phrase
- **Best for:** Expressive, emotive responses

---

## Latency Comparison

| Engine | Model | Latency | GPU |
|--------|-------|---------|-----|
| Kokoro | Kokoro-82M ONNX | ~0.5s | CPU |
| Chatterbox Turbo | chatterbox-turbo | ~2.1-2.4s | Radeon 8060S (ROCm) |
| Standard Chatterbox (benchmark) | chatterbox | ~10s | similar AMD hardware |

Turbo is ~5× faster than standard Chatterbox — achieved by reducing diffusion steps from 10 to 2.

---

## Voice Cloning

Chatterbox Turbo clones voice style from a reference WAV. The reference file is a ~10s sample of af_bella speech generated by Kokoro:

1. Generate a natural sentence via Kokoro (`af_bella` voice)
2. Save as `voices/female_default.wav`
3. Chatterbox loads it at startup — all synthesis uses this voice style

Result: both engines sound like the same persona, with Chatterbox adding prosody and emotion.

## ROCm on AMD Strix Halo

Docker + ROCm doesn't work here due to HIP library conflicts between the container and the host ROCm 7.2.1 installation. Chatterbox runs natively via systemd with these critical environment variables:

```bash
HSA_OVERRIDE_GFX_VERSION=11.5.1       # tells ROCm this is RDNA4 compatible
ROCR_VISIBLE_DEVICES=0                 # use iGPU
DEVICE=cuda                            # chatterbox expects "cuda" even on AMD
MIOPEN_FIND_MODE=FAST                  # skip exhaustive kernel search — critical for consistent latency
TORCH_ROCM_AOTRITON_ENABLE_EXPERIMENTAL=1
```

Without `MIOPEN_FIND_MODE=FAST`, first-run latency was 4-7s (variable) as MIOpen probed all kernel variants. With it: consistent 2.1-2.4s every time.
